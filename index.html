How computers work Khan Academy + Code.org

Explanation

A program is a list of instructions a computer follows one by one, very fast. There is no magic. A CPU understands a tiny set of operations add two numbers, move a value in memory, check if something is true and jump to a different instruction. When you write code, you write in a language designed for humans. A compiler or interpreter translates it to what the CPU actually runs.

Narrowing TypeScript handbook

Explanation

A type alias names a shape so you can reuse it: type User = { name: string; age: number }. Narrowing is how TypeScript figures out a more specific type inside an if block. After if (typeof x === "string"), TypeScript knows x is a string in that branch. A discriminated union a shared kind field with a literal type is the cleanest narrowing pattern and appears in every real codebase.

Task discriminated union

Define Circle with kind: "circle" + radius, Square with kind: "square" + side. Write getArea(shape: Circle | Square). Use kind to narrow. TypeScript knows which shape you have in each branch and blocks accessing radius in the square case.

Concept1.5 hrs

Resources
Docs
An overview of HTTP MDN

HTTP crash course Traversy Media

Explanation

A browser sends an HTTP request to a server. The server responds. The four methods: GET (read), POST (create), PUT/PATCH (update), DELETE (remove). Status codes tell you what happened: 200 (success), 201 (created), 400 (bad request), 401 (not authenticated), 404 (not found), 500 (server error). REST is a convention for designing URLs: GET /users returns all, GET /users/42 returns one, POST /users creates one.

Explanation

Three layers: a Controller handles HTTP receives requests, returns responses, no logic. A Service contains business logic database queries, calculations. A Module groups related controllers and services. This separation isn't arbitrary: it means you can test a service without a real HTTP request, swap implementations, and reason about the codebase by feature. Decorators declare intent: @Get("/users") turns a method into a GET handler syntactic sugar over the Express middleware patterns you already understand.

A DTO (Data Transfer Object) defines the shape of incoming data. With class-validator decorators (@IsEmail(), @IsString()) and NestJS's ValidationPipe, invalid requests are automatically rejected with a 400 error before reaching your service. Prisma in NestJS lives in a PrismaService inject it anywhere with constructor injection: constructor(private prisma: PrismaService). NestJS manages the instance.

The problem what and why
Capstone

The real problem this solves

In most cities and towns, residents have no easy way to report local issues broken streetlights, potholed roads, illegal dumping, flooding, unsafe walkways and no way to know if anyone else reported the same thing or if it's being addressed. Reports go to WhatsApp groups, Facebook posts, or nowhere. No accountability, no visibility, no resolution tracking.

LocalPulse is a community-run issue tracker. Residents submit and upvote problems in their area. Issues are mapped geographically. Moderators update statuses. Anyone can see what's been reported, what's being worked on, what's resolved. Deployable by any community that wants it. This is a real gap apps like Fix My Street exist in the UK, but most of sub-Saharan Africa and large parts of Asia and Latin America have nothing equivalent.

Architecture

Frontend: Next.js 15 with App Router, Tailwind CSS, Auth.js (Google OAuth + email). Backend: NestJS with Prisma + PostgreSQL, deployed as a separate service. Database: Primary PostgreSQL managed through Prisma for NestJS. Drizzle is used specifically in the analytics module so you have real side-by-side ORM experience in one project.
